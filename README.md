# 筋トレ管理

ボディメイクのためのトレーニング・カロリー記録アプリ（スマホ向けPWA）。
データはすべて端末のブラウザ内（IndexedDB）に保存され、サーバーは不要です。

**公開URL: https://hondak00.github.io/kintore-kanri/** （スマホで開いて「ホーム画面に追加」）
デプロイ手順は [DEPLOY.md](DEPLOY.md) を参照。

## 機能

- **ホーム** — 1日のカロリー収支（摂取 −〔基礎代謝×活動レベル＋実測運動消費〕）、PFC達成バー、体重・体脂肪率・運動消費のクイック記録
- **筋トレ記録** — ルーティン（胸の日など）からワンタップで開始、セットごとに重量×レップ数を記録。前回の記録を表示し「前回コピー」も可能
- **食事記録** — 朝昼夕間食ごとに品名・kcal・PFCを記録。一度入力した食品は「マイ食品」に保存され次回から検索→ワンタップ追加
- **記録表** — 縦＝種目、横＝日付のマトリクスで全セットの推移を一覧（ルーティン切替・セルタップで編集へ）
- **分析** — 体重/体脂肪率の推移、種目ごとのトップ重量・推定1RM（Epley式）、摂取vs消費グラフ
- **設定** — プロフィール（基礎代謝はMifflin-St Jeor式で自動計算）、目標設定、種目・ルーティン・マイ食品の管理、JSONバックアップ/復元

## 開発

```bash
npm install
npm run dev      # 開発サーバー
npm run build    # プロダクションビルド（dist/）
npm run preview  # ビルド結果の確認サーバー
npm run icons    # アプリアイコンPNGの再生成（依存ライブラリ不要）
```

動作確認（要 Google Chrome）:

```bash
npm run preview &
node scripts/smoke.mjs   # 全画面を操作して .screenshots/ にスクリーンショットを保存
```

## スマホで使う（ホーム画面に追加）

PWAのインストールには **HTTPS** が必要です。`dist/` は静的ファイルなので、
Cloudflare Pages / Netlify / Vercel などの無料静的ホスティングに置くのが簡単です。

1. `npm run build` で `dist/` を生成してホスティングにデプロイ
2. スマホのブラウザで開く
3. iPhone: 共有 →「ホーム画面に追加」 ／ Android: メニュー → 「アプリをインストール」

オフラインでも動作します（Service Workerが自動でキャッシュ）。

## データについて

- データは**この端末のブラウザ内のみ**に保存されます。別端末とは同期されません
- ブラウザのサイトデータ削除で消えるため、**設定 → データ管理 → バックアップ書き出し** を定期的に行ってください
- 機種変更時はバックアップJSONを書き出し、新端末で「バックアップから復元」

## 技術構成

| 項目 | 採用技術 |
|---|---|
| フレームワーク | Vite + React 19 + TypeScript |
| データ保存 | IndexedDB（Dexie 4 + dexie-react-hooks） |
| スタイリング | Tailwind CSS 4（ライトテーマ） |
| グラフ | Recharts 3（分析画面のみ遅延読み込み） |
| PWA | vite-plugin-pwa（autoUpdate） |
